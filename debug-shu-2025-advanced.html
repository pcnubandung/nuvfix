üîß Kemungkinan Penyebab & Solusi

Penyebab 1: Tidak Ada Data Transaksi 2025

Gejala: Semua menu transaksi kosong untuk tahun 2025 Solusi: 1. Input data penjualan tahun 2025 2. Input data pendapatan lain tahun 2025 3. Input data pengeluaran tahun 2025 4. Pastikan tanggal transaksi benar (tahun 2025)

Penyebab 2: Data Ada Tapi tahun_pembukuan Salah

Gejala: Ada data transaksi tapi SHU tetap 0 Kemungkinan: Data memiliki tahun_pembukuan = NULL atau != 2025 Solusi: 1. Buka Developer Tools (F12) 2. Tab Console ‚Üí ketik: console.log('Checking data...') 3. Periksa response API saat klik "Hitung SHU"

Penyebab 3: Biaya Operasional ‚â• Total Pendapatan

Rumus SHU: SHU = (Total Penjualan + Total Pendapatan Lain) - Biaya Operasional Jika hasil ‚â§ 0, maka SHU = 0 Contoh Masalah: - Total Penjualan: Rp 100,000,000 - Total Pendapatan Lain: Rp 10,000,000 - Total Pendapatan: Rp 110,000,000 - Biaya Operasional: Rp 120,000,000 - SHU = 110,000,000 - 120,000,000 = -10,000,000 ‚Üí Dianggap 0

Penyebab 4: Komponen SHU Belum Diatur

Gejala: Total persentase komponen SHU ‚â† 100% Solusi: 1. Menu SHU ‚Üí pilih tahun 2025 2. Klik "Atur Komponen SHU" 3. Set persentase total = 100% 4. Contoh: Cadangan 40%, Jasa Simpanan 25%, Jasa Transaksi 15%, dll 5. Simpan komponen

üßÆ Kalkulator Manual SHU

Hitung manual berdasarkan data yang Anda temukan:

Komponen Jumlah (Rp) Keterangan
Total Penjualan 2025 _____________ Dari menu Transaksi ‚Üí Penjualan
Total Pendapatan Lain 2025 _____________ Dari menu Transaksi ‚Üí Pendapatan Lain
TOTAL PENDAPATAN _____________ Penjualan + Pendapatan Lain
Total Pengeluaran 2025 _____________ Dari menu Transaksi ‚Üí Pengeluaran
Minus: Pembelian Barang _____________ Tidak masuk biaya operasional
Minus: Pembelian Aset _____________ Tidak masuk biaya operasional
BIAYA OPERASIONAL _____________ Pengeluaran - Pembelian Barang - Pembelian Aset
SHU TAHUN BERJALAN _____________ Total Pendapatan - Biaya Operasional
Analisis Hasil: - Jika SHU > 0: Masalah di backend/database - Jika SHU = 0: Pendapatan = Biaya Operasional - Jika SHU < 0: Rugi (sistem set ke 0)

üîç Debug Backend API

Cara Periksa Response API:

  1. Buka Developer Tools (F12)
  2. Tab Network
  3. Klik "Hitung SHU" di menu SHU
  4. Cari request ke /api/shu/hitung/2025
  5. Klik request tersebut ‚Üí tab Response
Response yang diharapkan: { "message": "SHU berhasil dihitung", "keuntunganBersih": 150000000, "totalAnggota": 50 } Response bermasalah: { "message": "SHU berhasil dihitung", "keuntunganBersih": 0, "totalAnggota": 0 }
